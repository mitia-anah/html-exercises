All elements

<a>

The anchor element (<a>) is one of the fundamental elements in HTML.

It allows a user to click on the link text it provides, and navigates to that link's location.

<abbr>

The abbreviation element ( <abbr>) is used to define the abbreviation or short form of an element.

It is represented as a series of letters.

<article>

the article element (<article>)is used to represent, a section of a page that consists of a composition that forms an independent part of a document, magasine, page or site.

<b>

The bold element(<b>) is used for formatting output, like bold text.

To make text bold in HTML, use the "b"tag.

<br>

The HTML
element (<br>)produces a line break in text (carriage-return).

It is useful for writing a poem or an address, where the division of lines is significant.

<body>

The body element(<body>) represents the main content of the document.

<button>

The button element( <button>) represents a button. If the element is not disabled, then the user agent should allow the user to activate the button.

<caption>

The caption element(<caption>) represents the title of the table that is its parent. If it has a parent and that is a table element.

<col>

The col element represents one or more columns in the column group represented by colgroup.

<dd>

The dd element( <dd>) represents the description, definition, or value, part of a term-description group in a description list (dl element).

<div>

The div element(<div>) has no special meaning at all. It represents its children. It can be used with the class, lang, and title attributes to mark up semantics common to a group of consecutive elements.

<dl>

The DL element(<dl>)Represents a description list consisting of zero or more name-value groups. Each group must consist of one or more names (dt elements) each followed by one or more values (dd elements).

<dt>

The DT element(<dt>) represents the term, or name, part of a term-description group in a description list (dl element).

<Footer>

The footer element(<Footer>)represents the "footer" of a document or section of a document. The footer element typically contains metadata about its enclosing section, such as who wrote it, links to related documents, copyright data, etc.

<H1-H6>

The H1-H6 elements(<H1-H6>)epresent headings and subheadings. These elements rank in importance according to the number in their name. The h1 element is said to have the highest rank, the h6 element has the lowest rank, and two elements with the same name have equal rank.

<Head>

The head element(<Head>) represents a collection of metadata for the Document.

<Header>

The Header lement(<Header>)represents the "header" of a document or section of a document. The header element is typically used to group a set of h1–h6 elements to mark up a page's title with its subtitle .

<Htlm>

The html element(<Htlm>) represents the root of an HTML document.

<Ifram>

The iframe element( <Ifram>) represents a nested browsing context.

<Image>

An img element(<Image>) represents an image. The image given by the src attribute is the embedded content, and the value of the alt attribute is the img element's fallback content.

<Li>

The li element(<Li>) represents a list item.

<Link>

The link element(<Link>) allows authors to link their document to other resources.

<Main>

The main element( <Main>) is an exact analogue of ARIA's role="main", and is designed to show screenreaders and assistive technologies exactly where main content begins,

<Meta>

The meta element(<Meta>) represents various kinds of metadata that cannot be expressed using the title, base, link, style, and script elements.

<Nav>

The navigation element(<Nav>)represents navigation for a document. The nav element is a section containing links to other documents or to parts within the current document.

<Ol>

The ol element(<Ol>) represents a list of items, where the items have been intentionally ordered, such that changing the order would change the meaning of the list.

<Option>

The option elemen(<Option>) represents an option in a select element or as part of a list of suggestions in a datalist element.

<P>

The p element(<P>) represents a paragraph.

<Script>

The script element(<Script>) allows authors to include dynamic script and data blocks in their documents.

<Section>

The section element(<Secion>)represents a generic document or application section. In this context, a section is a thematic grouping of content, typically with a header, possibly with a footer.

<Style>

The style element(<Style>) allows authors to embed style information in their documents. The style element is one of several inputs to the styling processing model.

<Summary>

The summary element(<Summary>) represents a summary, caption, or legend for the rest of the contents of the summary element's parent details element, if any.

<Table>

The table element(<Table>) represents data with more than one dimension, in the form of a table.

<Tbody>

The tbody element(<Tbody>) represents a block of rows that consist of a body of data for the parent table element, if the tbody element has a parent and it is a table.

<Td>

The td element(<Td>) represents a data cell in a table.

<Tfoot>

The tfoot element(<Tfoot>) represents the block of rows that consist of the column summaries (footers) for the parent table element, if the tfoot element has a parent and it is a table.

<Th>

The th element(<Th>) represents a header cell in a table.

<Thead>

The thead element(<Thead>) represents the block of rows that consist of the column labels (headers) for the parent table element, if the thead element has a parent and it is a table.

<Title>

The title element(<Title>) represents the document's title or name.

<Tr>

The tr element( <Tr>) represents a row of cells in a table.

<Ul>

The ul element(<Ul>) represents a list of items, where the order of the items is not important — that is, where changing the order would not materially change the meaning of the list.
